# 1. Noise Distribution Validation

## Experiment Goal

Verify that the **latent noise distribution after embedding still follows a standard Gaussian distribution**, which is critical because diffusion models assume the latent follows ( N(0,1) ).

## Observed Results

Latent statistics:

* Latent shape: **[1, 4, 64, 64]**
* Mean: **0.0008197**
* Variance: **0.9905336**

The histogram of encoded latent values closely follows the theoretical **(N(0,1))** probability density curve.

## Interpretation

The embedding strategy **does not distort the overall latent distribution**.
Even after inserting payload bits into selected latent positions using truncated Gaussian tails, the **global statistics remain nearly identical to the expected diffusion prior**.

## Key Insight

This confirms that:

* The embedding process remains **statistically stealthy**
* The diffusion model can still **generate realistic images**
* The payload embedding does not introduce detectable distribution shifts

This property is important for **steganographic security**, because large deviations in noise statistics could reveal the presence of hidden data.

---

# 2. Message vs Non-Message Latent Distribution

## Experiment Goal

Compare the noise distribution at:

* **payload embedding positions (Cardan grille indices)**
* **non-payload positions**

## Observed Pattern

The plot shows:

* **Message positions** are pushed into **extreme Gaussian tails**
* **Non-message positions** remain centered around zero

This separation clearly distinguishes:

* payload-carrying latent values
* normal diffusion latent values

## Interpretation

Embedding is achieved by pushing payload bits into **tail regions of the Gaussian distribution**.

This preserves the **overall noise distribution** while still encoding recoverable information.

The key property here is **localized distribution modification**, rather than global distortion.

---

# 3. Tail Separation for Bit Encoding

## Experiment Goal

Validate that **bit values are encoded using opposite Gaussian tails**.

Encoding scheme:

* **bit = 0 → negative tail**
* **bit = 1 → positive tail**

## Observed Results

Bit distribution means:

* bit=0 mean: **−2.78075**
* bit=1 mean: **2.86259**

## Interpretation

These means lie well beyond the threshold region (k ≈ 2.5), meaning:

* encoded values are strongly separated
* bit classification becomes very reliable

This large separation ensures that even after diffusion generation and inversion, the sign of recovered values remains stable.

## Key Insight

Tail separation makes **sign-based decoding robust**.

Because:

[
\text{bit} = sign(x_T)
]

This is the fundamental mechanism enabling payload recovery.

# 4. Latent Recovery After DDIM Inversion

## Experiment Goal

Verify whether **DDIM inversion successfully reconstructs the original latent noise** used during generation.

## Observed Statistics

Original latent:

* Mean = **0.0008197**
* Variance = **0.9905336**

Recovered latent:

* Mean = **−0.0008383**
* Variance = **0.9775391**

## Scatter Plot Analysis

The scatter plot comparing **original vs recovered latent values** shows two clear clusters:

* negative cluster for bit=0
* positive cluster for bit=1

## Error Histogram

Absolute error distribution shows:

* most errors are **very small (< 0.5)**
* few larger deviations

## Sign Consistency

Sign consistency at Cardan grille indices:

**1.0 (100%)**

## Interpretation

Even though inversion introduces small numeric errors, **sign preservation remains perfect**.

Since decoding depends only on sign, not magnitude, payload recovery remains extremely reliable.

# 5. ECC Overhead Analysis

## Purpose

Evaluate the trade-off between **payload redundancy and robustness**.

## Results

| ECC Mode     | Original Bits | Encoded Bits | Redundancy Ratio | Overhead |
| ------------ | ------------- | ------------ | ---------------- | -------- |
| None         | 200           | 200          | 1.0              | 0%       |
| Rep3         | 200           | 600          | 3.0              | 200%     |
| Hamming(7,4) | 200           | 350          | 1.75             | 75%      |

## Interpretation

ECC increases robustness by adding redundancy.

However, this reduces payload capacity.

Tradeoff:

* **Higher redundancy → stronger error correction**
* **Lower capacity**

# 7. ECC Robustness Under Distortion

## Experiment

Apply distortions and compare decoding performance.

### Without ECC

| Distortion  | BER       | Errors |
| ----------- | --------- | ------ |
| clean       | 0.0       | 0      |
| jpeg_q60    | **0.035** | 7      |
| blur_r1.0   | 0.0       | 0      |
| resize_0.75 | 0.0       | 0      |

### With Rep3 ECC

| Distortion  | BER       | Errors |
| ----------- | --------- | ------ |
| clean       | 0.0       | 0      |
| jpeg_q60    | **0.005** | 1      |
| blur_r1.0   | 0.0       | 0      |
| resize_0.75 | 0.0       | 0      |

## Interpretation

ECC dramatically improves robustness.

Example:

JPEG compression:

* BER reduced from **0.035 → 0.005**
* errors reduced from **7 → 1**

This shows ECC can **correct many bit errors introduced by image distortions**.
